# 0002 PostgreSQL 作为业务数据权威来源

状态：Accepted

日期：2026-07-08

## 背景

FilmOS 需要保存订单、配方、客户、机器、生产任务、用户和审计数据。这些数据存在关联、唯一性、状态一致性和事务要求。

## 决策驱动因素

- 多实体业务操作需要 ACID 事务；
- 订单、任务和基础数据具有明确关系；
- 需要数据库约束、迁移和可靠查询；
- 团队已经采用 SQLAlchemy 与 Alembic。

## 考虑过的方案

### PostgreSQL

支持关系、事务、JSONB、索引和成熟迁移工具，适合作为核心业务数据库。

### Redis 或其他非关系存储作为核心数据库

适合缓存或特定状态，但不适合作为当前关系型业务数据的唯一权威来源。

## 决策

- PostgreSQL 是 FilmOS 业务数据的权威来源；
- SQLAlchemy Model 表达应用数据模型；
- Alembic migration 是生产数据库结构变更的唯一正式渠道；
- Redis 只能承担明确的缓存、临时状态或会话用途，不能成为订单与生产任务正确性的唯一依赖。

## 结果与代价

### 正面影响

- 可以用事务保证订单与生产任务一致；
- 可以用外键、唯一约束和检查约束守护数据；
- 数据结构变更可审查和追踪。

### 负面影响

- migration 需要持续维护和验证；
- 查询和索引需要随数据规模优化；
- 高并发编号和队列操作仍需显式设计锁与冲突处理。

### 后续工作

- CI 增加空数据库完整 migration 验证；
- 为核心查询增加必要索引；
- 将跨实体看板操作收口为后端原子事务。
