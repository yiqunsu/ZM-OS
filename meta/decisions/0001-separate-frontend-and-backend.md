# 0001 前后端分离，FastAPI 为唯一业务入口

状态：Accepted

日期：2026-07-08

## 背景

FilmOS 是从原 Next.js 全栈单体应用重写而来。订单、排产、基础数据和 AI 操作需要共享同一套业务规则，并能够独立测试和演进。

## 决策驱动因素

- 防止 UI、数据库访问和业务规则相互耦合；
- 让手工页面和 AI 助手复用同一套业务能力；
- 支持前后端独立部署、测试和扩容；
- 使用 Python 生态承载后端业务和 AI 能力。

## 考虑过的方案

### 保留 Next.js 全栈单体

部署简单，但业务规则容易分散在页面、Route Handler 和数据库调用中，也不利于 Python Agent 与业务逻辑复用。

### Next.js 前端 + FastAPI 后端

增加 API、鉴权和跨服务配置成本，但职责清晰，符合本项目长期方向。

## 决策

- Next.js 只承担 UI、登录交互和后端 API 调用；
- FastAPI 是 REST、业务规则和数据访问的唯一入口；
- 前端不得直连 PostgreSQL；
- 手工操作和 AI 写操作必须调用相同的后端业务能力。

## 结果与代价

### 正面影响

- 业务规则可以集中测试；
- 前后端可以独立部署；
- AI 与页面不需要维护两套规则；
- 数据库访问边界清楚。

### 负面影响

- 需要维护 API 契约、JWT 和 CORS；
- 前后端类型可能发生漂移；
- 本地开发需要同时运行多个服务。

### 后续工作

- 保持 Router 薄、业务规则集中在 Service；
- 逐步从 OpenAPI 生成前端类型；
- 通过同源代理或明确的 CORS 配置完善生产部署。
